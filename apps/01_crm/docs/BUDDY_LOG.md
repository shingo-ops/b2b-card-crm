## 1. 記録の定義（5W2H）

### 1.1 いつ（When）

| タイミング | 内容 | 担当 |
|------------|------|------|
| **毎日** | 会話ログをスプレッドシートに自動記録 | GASトリガー |
| **毎週金曜 18:00** | ログ収集 → Discord通知 → 壁打ち | GAS + Human + Claude Code |
| **壁打ち後** | 成長記録をBUDDY_LOG.mdに追記 | Claude Code |
| **バージョンアップ時** | 仕様変更をコミット | Claude Code（Human承認後） |

### 1.2 誰が（Who）

| 役割 | 担当 | 責務 |
|------|------|------|
| **ログ収集** | GASトリガー（自動） | 会話データの蓄積 |
| **Discord通知** | GAS + Webhook（自動） | 壁打ちMTGの招集 |
| **壁打ち** | Human + Claude Code | Good/More/Bad/禁止項目の決定 |
| **メンテナンス実行** | Claude Code | Buddy仕様の更新（Human承認後） |
| **効果測定** | Claude Code | 数値分析、Before/After比較 |

### 1.3 どこで（Where）

| 場所 | 用途 | 形式 |
|------|------|------|
| **スプレッドシート「Buddy対話ログ」** | 生データ保存 | 行データ |
| **BUDDY_LOG.md** | バージョン定義・成長サマリー | Markdown |
| **Discord** | 壁打ちMTG通知 | Webhook |
| **ターミナル** | 壁打ち実施 | Claude Code会話 |

### 1.4 何を（What）

| 記録項目 | 形式 | 例 |
|----------|------|-----|
| **バージョン番号** | vX.Y.Z | v1.0.0 |
| **会話ログ** | 日時, ユーザー, 入力, 応答, 評価 | スプレッドシート行 |
| **週次指標** | 数値 | 会話数: 45, 完了率: 82% |
| **壁打ち結果** | 5W2H | Good/More/Bad/禁止 |
| **メンテナンス内容** | 5W2H + diff | 変更前後の差分 |

### 1.5 どうやって（How）

```
週次PDCAループ:

① GASトリガー（毎日）
   └── 会話ログをスプレッドシートに蓄積

② 週次トリガー（金曜 18:00）
   └── clasp run collectBuddyLogs()
       ├── ログ集計（数値化）
       └── BUDDY_LOG.md に週次サマリー追記

③ Discord通知
   └── webhook で壁打ちMTG招集

④ 壁打ち（Human + Claude Code）
   ├── 今週のログレビュー
   ├── Good Point 抽出
   ├── More Point 抽出
   ├── Bad Point / 禁止項目 決定
   └── 次週の改善方針決定

⑤ メンテナンス実行
   ├── Buddy仕様更新
   ├── メッセージ文言調整
   └── clasp push

⑥ 効果測定（次週）
   └── Before/After 数値比較
```

### 1.6 なぜ（Why）

| 目的 | 理由 |
|------|------|
| **Buddyのパフォーマンス最大化** | 担当者の成果向上 → 組織の成果向上 |
| **継続的改善** | PDCAループで質と精度を向上 |
| **再現性の確保** | 成功パターンを他プロジェクトに横展開 |
| **バージョン管理** | いい/悪いの判断材料、ロールバック可能 |

### 1.7 どれぐらい（How Much）

| 指標 | 目標値 | 測定方法 | 現在値 |
|------|--------|----------|--------|
| **週次会話数** | ベースライン設定中 | ログカウント | - |
| **リマインド完了率** | 80%以上 | 完了/全リマインド | - |
| **ポジティブ反応率** | 70%以上 | キーワード検出 | - |
| **壁打ち実施率** | 100% | 実施/予定 | - |
