## 2. 目的定義（5W2H）

### 2.1 プロジェクトゴール

| 項目 | 内容 |
|------|------|
| **いつ** | CRM利用開始時〜継続的に |
| **誰が** | 営業担当者（主要ユーザー） |
| **どこで** | CRMダッシュボード上 |
| **何を** | 「開きたくなる・閉じたくなくなる・ずっと使いたくなる」体験 |
| **どうやって** | 依存性心理を活用したUI/UX設計 |
| **なぜ** | 金銭報酬＋働きがい＋居場所の提供→離職率低下・パフォーマンス向上 |
| **どれぐらい** | 離職率: 測定後に目標設定、DAU: 80%以上を目標 |

### 2.2 ユーザーに提供する価値

| 価値 | 定義 | 測定方法 |
|------|------|----------|
| **快感** | タスク完了時・目標達成時に達成感を感じる | 完了率、継続率 |
| **承認欲求充足** | 他者・Buddyから認められる感覚 | 称賛受信数、バッジ獲得数 |
| **居場所認識** | 「ここに自分の場所がある」という感覚 | ログイン頻度、滞在時間 |

### 2.3 制約条件（禁止事項）

| 禁止事項 | 理由 | 違反時の影響 |
|----------|------|--------------|
| 音声・効果音の使用 | 環境によっては不適切（会議中等） | ユーザー離脱 |
| 過度なポップアップ | ストレス要因 | UX悪化 |
| 強制的な操作要求 | 離脱要因 | 利用率低下 |
| 意味のないアニメーション | 疲労感 | 滞在時間低下 |

## 3. 設計思想（5W2H）

### 3.1 依存度公式

```
依存度 = (継続 × 成果) × (愛着 + 心理フック + 収集欲) × 体験品質
```

| 変数 | 定義 | 数値目標 | 実装手段 |
|------|------|----------|----------|
| **継続** | 毎日触れる習慣 | 連続ログイン7日以上: 50%ユーザー | ログインボーナス、ストリーク |
| **成果** | 進捗・達成の可視化 | 週1回以上目標確認: 80%ユーザー | レベル、ポイント、グラフ |
| **愛着** | Buddyとの感情的つながり | Buddy好感度調査: 4.0/5.0以上 | 人格設定、感情システム |
| **心理フック** | 予測不能な報酬 | レア獲得時のセッション延長: +30% | 変動報酬、ガチャ |
| **収集欲** | コレクター心理 | バッジ収集率平均: 40%以上 | バッジ図鑑、レアリティ |
| **体験品質** | 視覚的喜び・没入感 | UI満足度調査: 4.0/5.0以上 | アニメーション、色彩 |

### 3.2 最重要原則

| 原則 | 定義 | 根拠 |
|------|------|------|
| **関係 > 機能** | ツールは代替可能だが感情的つながりは代替不可能 | アニメ・ゲームの成功要因分析より |
| **Buddyが核** | 「このBuddyと一緒に成長したい」という感情を生む | パラソーシャル関係の心理学 |

## 5. 機能仕様（5W2H）

### 5.1 変動報酬システム

#### 5.1.1 ラッキーリード

| 項目 | 内容 |
|------|------|
| **いつ** | 毎日AM8:00にリセット、ログイン時に表示 |
| **誰が** | 全営業担当者 |
| **どこで** | ダッシュボード・Buddyカード内 |
| **何を** | ランダムで1件のリードを「今日のラッキーリード」として表示 |
| **どうやって** | 担当リードからランダム選出（温度感「高」優先率60%） |
| **なぜ** | 変動報酬による予測不能な喜び、特定リードへの注目誘導 |
| **どれぐらい** | 該当リード成約時: 通常の2倍ポイント（200pt） |

**受入条件**:
- [ ] 毎日異なるリードが選出される
- [ ] 担当リードが0件の場合は非表示
- [ ] ラッキーリード成約時にボーナスポイントが付与される

#### 5.1.2 Buddy反応ガチャ

| 項目 | 内容 |
|------|------|
| **いつ** | タスク完了時（リード連絡、商談メモ記入、成約等） |
| **誰が** | 全ユーザー |
| **どこで** | Buddyメッセージエリア |
| **何を** | 完了時のBuddyメッセージがレアリティ別にランダム変化 |
| **どうやって** | 乱数生成でレアリティ判定→対応メッセージ表示 |
| **なぜ** | 毎回同じ反応では飽きる。予測不能性がドーパミンを刺激 |
| **どれぐらい** | 下記レアリティ表参照 |

| レアリティ | 出現率 | メッセージ例 |
|------------|--------|--------------|
| ノーマル | 70% | 「いい調子だね！」「ナイス！」 |
| レア | 20% | 「実は...君のこと尊敬してるんだ」 |
| スーパーレア | 8% | 「今日の君、いつもより輝いてるよ！」 |
| ウルトラレア | 2% | 「ここだけの話...君が一番頑張ってるって思ってる」 |

**受入条件**:
- [ ] 完了アクションごとにランダム判定が行われる
- [ ] レアリティに応じてメッセージスタイルが変化（色・アイコン）
- [ ] ウルトラレア出現時は特別エフェクト（紙吹雪等、音なし）

#### 5.1.3 隠しバッジ

| 項目 | 内容 |
|------|------|
| **いつ** | 条件達成時に即時 |
| **誰が** | 条件を満たしたユーザー |
| **どこで** | バッジ獲得通知→バッジコレクション |
| **何を** | 条件非公開のシークレットバッジ |
| **どうやって** | バックエンドで条件判定、達成時にポップアップ |
| **なぜ** | 狩猟本能の刺激、「発見した！」という喜び |
| **どれぐらい** | 全10種類（初期）、条件例: 月曜AM8時台ログイン3回連続 |

**受入条件**:
- [ ] バッジ図鑑では「？？？」と表示
- [ ] 獲得時に「シークレットバッジ獲得！」通知
- [ ] 獲得後に条件がヒントとして表示される

#### 5.1.4 週末ボーナス抽選

| 項目 | 内容 |
|------|------|
| **いつ** | 毎週金曜18:00に抽選、結果は月曜ログイン時に表示 |
| **誰が** | 週間活動があったユーザー |
| **どこで** | ログイン時ポップアップ |
| **何を** | 週間活動量に応じたポイント抽選 |

| **どれぐらい** | 1〜100pt（活動量に応じて上限変動） |

| 週間活動量 | 抽選範囲 |
|-----------|----------|
| 1-10アクション | 1-20pt |
| 11-30アクション | 5-50pt |
| 31アクション以上 | 10-100pt |

**受入条件**:
- [ ] 金曜18:00に自動抽選が実行される
- [ ] 活動0のユーザーは抽選対象外
- [ ] 月曜ログイン時に結果表示（スロット風演出、音なし）

#### Human発言記録（原文）

1. 「UI/UX開発の定義として担当者が開きたくなる、閉じたくなくなる、ずっと使っていたくなるような仕様に仕上げて欲しい」

2. 「収集が大変なほど手に入れた時の快感も高いと感じるはず」

3. 「仕事なので金銭の報酬が優先的な成果だがプラスアルファで働きがいを感じる居場所を提供すれば付加価値がつき離職率は抑えられるはず」

4. 「短期的な報酬もあるし会社のMVPによる大きなゴールを掲げるのも効果があると思う」

6. 「人によって承認されたいポイントも快適だと思う環境も異なるのでパターン別の提供方法も必要」

7. 「依存度=継続x成果に加えて愛着や人間的な心理的要素が高いと考えている」

8. 「音は人によっては環境面で無音がいい場合もあるので音系の演出はなし」

         ↓
離脱率低下 + エンゲージメント向上
