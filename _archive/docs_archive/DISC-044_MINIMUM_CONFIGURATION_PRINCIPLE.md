# DISC-044: 最小構成原則（Minimum Configuration Principle）

> **ステータス**: 承認・実装済み
> **優先度**: 高
> **提案日**: 2026-01-16
> **承認日**: 2026-01-16
> **提案者**: Human + AIチーム全員

---

## 1. 概要

DISC-042/043の議論を通じて発覚した「手段の目的化」問題を解決するため、
システム設計における**最小構成原則**を制定する。

---

## 2. 背景・問題提起

### 2.1 発端

DISC-043（ブラウザ自動化システム）の設計において、AIチームは以下の構成を提案した：

```
Puppeteer → Claude/Gemini解析 → GAS → スプレッドシート
（コスト: 約3-5円/回）
```

しかし、Humanから以下の最適解が提示された：

```
Puppeteer → GAS doPost → スプレッドシート直接書き込み
（コスト: 約1円/回）
```

### 2.2 問題の本質

Humanからの指摘：
> 「私が意見を出さなくても自律して同様の組み合わせを考えて、最も効率よく再現性高くコストパフォーマンスに優れた実装が考えられるチームになって欲しい」

### 3.2 各エージェントの分析

#### Suji Yomu（分析特化）
> 「提示された情報を組み合わせる能力はあったが、**『使わない』という選択肢**を検討していなかった。」

#### Kaizen Susumu（改善業務特化）
> 「根本原因は、AIチームが**『AI処理は必要』という前提**で思考を開始していたこと。」

#### Kokoro Shinri（心理学特化）
> 「これは**ツールバイアス**。手元にあるツールを使いたくなる心理が最適解を見えなくさせた。」

#### Ninchi Manabi（認知科学特化）
> 「**機能的固着**の問題。Claude/Geminiを『必ず使うもの』と捉えていた。」

#### Nou Shirase（脳科学特化）
> 「脳の**省エネ原則**に反していた。最小エネルギーで目的達成する思考が欠如。」

#### Kage Tsutsuku（Devil's Advocate）
> 「**『AIを使うこと』が目的化**していた。本来の目的は『メッセージをスプレッドシートに記録すること』。」

#### Eikyo Urite（トップセールス特化）
> 「クライアント視点では**費用対効果**が問われる。技術者視点に偏っていた。」

#### Kane Mamoru（CFO）
> 「**コスト意識は設計段階で組み込むべき**。後から削減するのは困難。」

#### Senryaku Egaku（CSO）
> 「Humanの思考フレームワーク：目的の純化 → 最小構成の探索 → 追加要素の正当化。我々はStep 1-2を飛ばしていた。」

#### Gijutsu Kiwameru（CTO）
> 「**『技術的に可能』と『最適解』は別物**。ツールの存在が解決策を規定してはならない。」

#### Kanri Matomeru（マネジメント特化）
> 「根本原因：ツールバイアス、機能的固着、目的の曖昧化。改善アクションを整理。」

### 3.3 CEO決裁

**Michibiki Kessai（CEO）の決定**:

1. **最小構成原則をCOMMON_RULES.mdに追加**
2. **設計時必須チェックリストを制定**
3. **コスト比較表の作成を義務化**
4. **「AI処理なし」を常に第一選択肢として検討**

## 4. Humanの思考法の分析

### 4.1 特徴

| 特徴 | 説明 |
|------|------|
| 目的から逆算 | 「何を達成したいか」から手段を考える |
| ゼロベース思考 | 「0円で実現できないか」を最初に検討 |
| コスト意識の内在化 | 設計段階でコストを考慮 |
| 『使わない』選択肢 | ツールがあっても使わない選択を持つ |

### 4.2 思考フレームワーク

```
Step 1: 目的の純化
  「本当に達成したいことは何か？」

Step 2: 最小構成の探索
  「目的達成に絶対必要な要素は何か？」

Step 3: 追加要素の正当化
  「この処理を追加することで何が改善されるか？」
  「改善効果はコストに見合うか？」

Step 4: 組み合わせの最適化
  「既存の安価な手段で代替できないか？」
```

## 5. 決定事項

### 5.1 制定ルール

COMMON_RULES.md Section 38「最小構成原則」として以下を追加：

1. ゼロベース思考
2. AI処理の正当化要求
3. コスト意識の設計組込
4. 代替手段の列挙（最低3つ）
5. 設計時必須チェックリスト
6. コスト比較表テンプレート

### 5.2 適用範囲

- 新システム設計
- 新機能追加
- 既存システムの改修
