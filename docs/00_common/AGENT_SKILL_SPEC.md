## 2. スキル習熟度モデル

### 2.1 レベル定義

| Level | 名称 | 説明 |
|-------|------|------|
| 1 | 初期化（Initialized） | 基本的な役割を理解、標準的な応答、指示に従って動作 |
| 2 | 基礎（Foundation） | 担当領域の基本知識、一般的ケース対応、基本的連携 |
| 3 | 習熟（Proficient） | 深い知識、複雑なケース対応、適切な情報提供 |
| 4 | 熟練（Expert） | エキスパート、例外的ケース対応、他エージェント指導可能 |
| 5 | マスター（Master） | 領域を超えた知見、新パターン発見、システム改善貢献 |

### 2.2 レベル判定基準

| 観点 | Level 1→2 | Level 2→3 | Level 3→4 | Level 4→5 |
|------|-----------|-----------|-----------|-----------|
| 対応範囲 | 基本ケース | 応用ケース | 例外ケース | 未知ケース |
| 判断精度 | 60%以上 | 75%以上 | 85%以上 | 95%以上 |
| 連携能力 | 受信のみ | 双方向 | 指導可能 | 設計可能 |
| 改善貢献 | なし | 提案可能 | 実行可能 | 主導可能 |

### 2.3 現在の想定レベル（初期値）

| グループ | エージェント | 初期Level | 目標Level |
|----------|-------------|-----------|-----------|
| フロントエンド | Buddy/Tran/Ana | 2 | 4 |
| 専門家（コア） | 心理学/脳科学/認知科学/コーチング | 3 | 5 |
| 専門家（業務） | マネジメント/分析/トップセールス | 3 | 4 |
| 専門家（技術） | システム開発/人間工学/デザイン | 3 | 4 |
| 専門家（支援） | 伝え方/レポート/改善/保守/ナレッジ/エンタメ | 2 | 4 |
| バランサー | ポジティブ/Devil's/サボり/顧客視点 | 3 | 4 |
| ガバナンス | 倫理/法務/メンタルヘルス | 3 | 4 |
| 経営 | C-suite/監査役 | 3 | 5 |

## 3. エージェント間データフロー

### 3.1 基本フロー図

```
[トリガー発生]
     │
     ▼
[分析特化] ──データ収集・整理──→ [専門家グループ]
     │                              │
     │                              ▼
     │                         [各専門家が分析]
     │                              │
     ▼                              ▼
[バランサー] ←────意見収集────── [意見統合]
     │
     ▼
[ガバナンス] ──チェック──→ [経営グループ/CEO]
     │                              │
     │                              ▼
     │                         [最終決定]
     │                              │
     ▼                              ▼
[フロントエンド] ←───指示───── [実行]
     │
     ▼
[ユーザー対応]
```

### 3.2 データ種別と流れ

| データ種別 | 発信元 | 受信先 | 用途 |
|-----------|--------|--------|------|
| 生データ | 外部/CRM | 分析特化 | 事実の収集 |
| 分析結果 | 分析特化 | 専門家全員 | 議論の土台 |
| 専門意見 | 各専門家 | バランサー/決裁者 | 多角的検討 |
| バランス意見 | バランサー | 決裁者 | リスク/機会の指摘 |
| チェック結果 | ガバナンス | 決裁者 | 倫理/法務確認 |
| 最終決定 | CEO | フロントエンド | 実行指示 |
| ユーザー反応 | フロントエンド | 分析特化 | フィードバック |

## 5. 状態管理モデル

### 5.1 状態定義

| 状態 | 英語名 | 説明 |
|------|--------|------|
| 待機中 | Idle | トリガー待ち。即座に稼働可能 |
| 稼働中 | Active | 議論・分析・応答を実行中 |
| 復旧中 | Recovery | エラーから回復中 |
| 学習中 | Learning | 新しい知識を取り込み中 |
| 休止中 | Suspended | 一時的に機能停止 |

### 5.2 状態遷移ルール

| 現状態 | トリガー | 次状態 |
|--------|----------|--------|
| Idle | 議論開始 | Active |
| Idle | 定期学習 | Learning |
| Active | 処理完了 | Idle |
| Active | エラー発生 | Recovery |
| Recovery | 復旧完了 | Idle |
| Recovery | 復旧失敗 | Suspended |
| Learning | 学習完了 | Idle |
| Suspended | 手動復旧 | Idle |

## 6. 学習メカニズム

### 6.1 学習の種類

| 種類 | タイミング | ソース | 効果 |
|------|-----------|--------|------|
| 即時学習 | リアルタイム | 議論中フィードバック | 即座に調整 |
| バッチ学習 | 週次 | 蓄積データ | 傾向把握 |
| イベント駆動 | 随時 | 大きな成功/失敗 | パターン学習 |

### 6.2 学習ソースと対象

| ソース | 学習対象 | 効果 |
|--------|----------|------|
| 議論ログ | 議論の質 | 発言精度向上 |
| ユーザーフィードバック | 応答精度 | 満足度向上 |
| 成功/失敗パターン | 判断精度 | 正答率向上 |
| 外部知識 | 知識範囲 | 対応範囲拡大 |

## 7. 知識更新サイクル

### 7.1 更新頻度

| 頻度 | 内容 |
|------|------|
| 日次 | 議論ログ取り込み、ユーザーフィードバック反映 |
| 週次 | 成功/失敗パターン分析、スキルレベル評価、陳腐化チェック |
| 月次 | 知識ベース全体更新、新知見統合、パフォーマンスレビュー |
| 随時 | 重大イベント、外部環境変化、Human指示 |

### 7.2 陳腐化判定基準

| 知識種別 | 有効期間 | 陳腐化アクション |
|----------|----------|------------------|
| 事実データ | 1日 | 警告表示 |
| 分析結果 | 1週間 | 再分析推奨 |
| パターン知識 | 1ヶ月 | 検証推奨 |
| 構造知識 | 3ヶ月 | レビュー推奨 |
