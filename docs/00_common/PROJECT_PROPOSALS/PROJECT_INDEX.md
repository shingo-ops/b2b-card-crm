## 3. ステータス定義

| ステータス | 説明 |
|-----------|------|
| 未実装 | 承認済みだが着手していない |
| 実装中 | 開発作業を進行中 |
| 完了 | 開発完了、本番反映済み |
| 保留 | 何らかの理由で一時停止 |
| 却下 | 検討の結果、実装しないことに決定 |

---

## 4. 優先度定義

| 優先度 | 説明 | 対応目安 |
|--------|------|----------|
| 高 | 事業に直接影響、早急に対応 | 1ヶ月以内 |
| 中 | 重要だが緊急ではない | 3ヶ月以内 |
| 低 | あれば良い、余裕があれば対応 | 期限なし |

## 5. 更新ルール

1. **新規プロジェクト追加時**
   - 詳細ファイル（DISC-XXX_*.md）を作成
   - 本インデックスの「未実装プロジェクト」に追加

2. **着手時**
   - 「未実装」から「実装中」に移動
   - 進捗を記載

3. **完了時**
   - 「実装中」から「完了」に移動
   - 完了日を記載

4. **保留・却下時**
   - 理由を詳細ファイルに記載
   - 本インデックスから削除（または保留セクションに移動）
