# DISC-042: 公開コンテンツ情報収集AIエージェント

> **ステータス**: 未実装
> **優先度**: 中
> **提案日**: 2026-01-15
> **提案者**: Human + Claude Code

---

## 1. 概要

Web Search / WebFetch を活用して、公開されているブログ・公式サイト・ニュースサイトから
顧客に有益な情報を自動収集するAIエージェントシステム。

---

## 2. 背景・課題

- 顧客への提案ネタとなる最新情報を手動で収集するのは時間がかかる
- 複数のソースを定期的にチェックするのは非効率
- 重要な情報を見逃すリスクがある

---

## 3. 提案内容

### 3.1 機能

| 機能 | 説明 |
|------|------|
| ソース管理 | 監視対象サイトをスプレッドシートで管理 |
| 定期収集 | GASトリガーで定期的に情報取得 |
| 解析・要約 | Claude/Gemini で重要度判定・要約生成 |
| 記録 | CRMスプレッドシートに自動記録 |
| 通知 | 重要な更新を通知 |

### 3.2 対象コンテンツ

- 公式メーカーサイトの新着情報
- 業界ブログ記事
- Note記事
- ニュースサイト
- 技術ドキュメント

### 3.3 取得方法

| 方法 | 用途 | 安全性 |
|------|------|--------|
| Web Search | 検索・要約 | 最も安全 |
| WebFetch | 特定URL取得 | 安全 |
| RSS/Atom | フィード対応サイト | 最も安全 |
| GAS UrlFetchApp | 軽量な取得 | 安全 |

---

## 4. アーキテクチャ

```
┌─────────────────────────────────────────────────────────────┐
│  監視対象リスト（スプレッドシート）                           │
│  - URL, カテゴリ, 取得方法, 最終チェック日時                 │
└─────────────────────────────────────────────────────────────┘
                          │
                          ▼ GASトリガー（日次/週次）
┌─────────────────────────────────────────────────────────────┐
│  取得エンジン                                                │
│  - RSS対応: GAS UrlFetchApp                                 │
│  - 静的ページ: WebFetch                                      │
│  - 検索: Web Search                                          │
└─────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  解析エンジン（Claude API / Gemini API）                     │
│  - 新着判定                                                  │
│  - 重要度スコアリング                                        │
│  - 要約生成                                                  │
│  - 顧客関連性判定                                            │
└─────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  CRM記録 + 通知                                              │
│  - 新着情報シートに記録                                      │
│  - Slack/メール通知（オプション）                            │
└─────────────────────────────────────────────────────────────┘
```

---

## 5. 期待効果

| 効果 | 説明 |
|------|------|
| 時間削減 | 情報収集の自動化で週数時間の削減 |
| 網羅性向上 | 見逃しリスクの低減 |
| 提案力向上 | 最新情報に基づく提案が可能に |
| 顧客満足度 | タイムリーな情報提供 |

---

## 6. リスク・注意点

| リスク | 対策 |
|--------|------|
| サイト構造変更 | RSS優先、定期メンテナンス |
| 過剰アクセス | 適切な間隔設定 |
| 情報の正確性 | 人間によるレビュー体制 |
| コスト増 | 解析は必要最小限に |

---

## 7. 実装ステップ

1. **Phase 1**: RSS対応サイトから開始
2. **Phase 2**: WebFetch で静的ページ対応
3. **Phase 3**: Claude/Gemini 解析機能追加
4. **Phase 4**: 通知機能追加
