# 1. 概要

## 1.1 担当者カルテとは

担当者カルテは、営業担当者ごとの総合的な情報を一元管理するダッシュボードである。
リーダー/オーナーが担当者の状態を把握し、適切なサポートを行うための資料として機能する。

## 1.2 アクセス権限

| 役割 | アクセス |
|------|---------|
| オーナー | 全担当者のカルテを閲覧・編集可能 |
| リーダー | 自チームの担当者のカルテを閲覧可能 |
| システム管理者 | 全担当者のカルテを閲覧可能 |
| 営業 | 自分のカルテのみ閲覧可能（一部制限あり） |
| CS | アクセス不可 |

## 1.3 担当者に見せない情報

| 情報 | 理由 |
|------|------|
| モチベーションスコア | プレッシャーを与えないため |
| 感情分析結果 | 監視されている感覚を防ぐため |
| アラートステータス | 不安を与えないため |
| オーナーメモ | 管理者間の情報共有のため |

# 3. ペンタゴングラフ

## 3.1 5つの軸

| 軸 | 内容 | データソース |
|----|------|-------------|
| 営業力 | 成約率、商談進捗 | 商談データ |
| 行動力 | 出勤率、アクション完了率 | ログイン、タスク |
| 知識 | 教育スコア、テスト結果 | 教育ツール |
| 自己管理 | 振り返りの質、目標達成率 | レポート、目標 |
| コミュニケーション | 対話ログ、顧客フィードバック | Buddy、商談メモ |

## 3.2 スコア算出ロジック

### 営業力（0-100）
```
(成約率 × 0.5 + 目標達成率 × 0.3 + 商談進捗率 × 0.2) × 100
```

### 行動力（0-100）
```
(出勤率 × 0.5 + 定時率 × 0.3 + アクション完了率 × 0.2) × 100
```

### 知識（0-100）
```
(テストスコア × 0.6 + コース完了率 × 0.4) × 100
```

### 自己管理（0-100）
```
(振り返り提出率 × 0.4 + 振り返り質スコア × 0.3 + 目標設定率 × 0.3) × 100
```

### コミュニケーション（0-100）
```
(Buddy対話質スコア × 0.5 + 顧客対応評価 × 0.5) × 100
```

## 3.3 3段階の記録

| フェーズ | タイミング | 用途 |
|---------|-----------|------|
| 採用時 | 入社日 | 診断ツールの予測値 |
| 教育終了時 | 教育完了日 | 診断精度の検証 + 教育効果 |
| 現在 | 週次更新 | 成長の可視化 |

### グラフの凡例
```
─ ─ ─  採用時（点線）
━ ━ ━  教育終了時（破線）
━━━━━  現在（実線）
```

## 3.4 診断精度チェック

| パターン | 採用時→教育終了 | 分析 |
|---------|----------------|------|
| 乖離 ±10以内 | 一致 | 診断精度◎ |
| 乖離 +10以上 | 上方乖離 | 教育効果が高い or 診断が過小評価 |
| 乖離 -10以上 | 下方乖離 | 教育で課題発覚 or 診断が過大評価 |

# 4. モチベーション分析

## 4.1 スコア構成

| 指標 | 算出方法 | 重み |
|------|---------|------|
| 行動スコア | ログイン率、アクション完了率 | 20% |
| 成果スコア | 目標達成率、前月比 | 20% |
| 学習スコア | 教育ツール進捗、テストスコア | 15% |
| 自己認識スコア | 振り返りの質（AI分析） | 20% |
| 感情スコア | 会話ログの感情分析 | 25% |
| 総合 | 上記の加重平均 | 100% |

## 4.2 アラート基準

| レベル | 条件 | アクション |
|--------|------|----------|
| 🟢 良好 | 70点以上 | 通常対応 |
| 🟡 注意 | 50-69点 | Buddyが寄り添い強化 |
| 🔴 要対応 | 50点未満 | オーナー/リーダーにDiscord通知 |
| ⚫ 緊急 | 30点未満 or 急激な低下（-20以上） | 即時アラート |

## 4.3 感情分析（Gemini API）

```javascript
const prompt = `

【評価基準】

- ストレス度（1-10）: 困っている、焦っている表現
```

## 5.2 ログイン/ログアウト記録

### ログイン記録

- ページ読み込み時に自動記録

### ログアウト記録（推定含む）

- beforeunload イベントで記録を試みる
- 15分ごとのハートビート送信
- 30分以上無操作 → 最終操作時刻をログアウト時刻として記録

## 5.3 出勤率の計算

```javascript
function calculateAttendanceMetrics(staffId, month) {
  const shiftDays = getShiftDays(staffId, month);      // シフト日数
  const loginDays = getLoginDays(staffId, month);      // ログイン日数
  const onTimeLogins = getOnTimeLogins(staffId, month); // シフト開始30分以内のログイン数

  return {
    attendanceRate: (loginDays / shiftDays) * 100,     // 出勤率
    punctualityRate: (onTimeLogins / loginDays) * 100  // 定時率
  };
}
```
