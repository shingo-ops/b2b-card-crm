# 1. 概要

## 1.1 目的

振り返りレポートは、営業担当者が自分の現在地と目標までの乖離を認識し、課題を自分で書き出すことで、目標達成のために何が足りなくて何をしなければいけないかを自覚させる仕組みである。

## 1.2 レポート種類

| レポート | 頻度 | 目的 |
|---------|------|------|
| 商談レポート | 商談後 | 商談の振り返り、次アクションの明確化 |
| 週次レポート | 週末 | 1週間の振り返り、来週の目標設定 |
| 月次レポート | 月末 | 1ヶ月の振り返り、成長の確認 |

## 1.3 運用ルール

| ルール | 内容 |
|--------|------|
| 必須 | 商談後24時間以内に商談レポート提出 |
| 必須 | 毎週金曜日までに週次レポート提出 |
| 必須 | 毎月最終営業日までに月次レポート提出 |
| 未提出 | Buddyがリマインド → 翌日もなければDiscord通知 |

# 2. 商談レポート

## 2.1 フォームの流れ

1. 商談結果を選択
2. 商談結果に応じて表示項目が変わる
3. 商談中に入力済みのデータは自動読み込み
4. 追加情報を入力
5. 商談ログ（必須）を貼り付け
6. 保存
7. Buddyからフィードバック + 壁打ち提案

## 2.2 入力項目

### 基本情報

| 項目 | 種類 | 必須 | 自動読込 | 備考 |
|------|------|------|---------|------|
| 提出日 | 自動 | - | - | システム日付 |
| 商談開始日 | 日付 | ✅ | ✅ | CSがアサインした日 |
| 商談結果 | プルダウン | ✅ | - | 設定シート参照 |

### 商談結果の選択肢

| 選択肢 | 追加表示項目 |
|--------|-------------|
| 成約 | 成約したポイント |
| 失注 | こうしたらもっと良かった点 |
| 追客 | 次回アクション日、アクション内容 |
| 見送り | 再アプローチ日、見送り理由 |
| 対象外 | 対象外理由のみ（他は非表示） |

### 顧客情報

| 項目 | 種類 | 必須 | 自動読込 | 備考 |
|------|------|------|---------|------|
| 顧客の国 | プルダウン | ✅ | ✅ | 設定シート参照 |
| 取り扱い商材 | チェックボックス | ✅ | ✅ | 複数選択可、設定シート参照 |
| 販売先 | チェックボックス | ✅ | ✅ | 複数選択可、設定シート参照 |
| 信頼重視/価格重視 | プルダウン | ✅ | ✅ | 設定シート参照 |

### 信頼重視/価格重視の基準

| 選択肢 | 基準 |
|--------|------|
| 信頼重視 | 価格以外の話題（品質・納期・対応）で商談が進んでいた |
| 価格重視 | 終始価格の話しかしていない。他の話題を振っても価格に戻る |
| 不明 | 初回接触で判断できない |

### 取引規模

| 項目 | 種類 | 必須 | 自動読込 | 備考 |
|------|------|------|---------|------|
| 1回の発注金額 | 数値 | ✅ | ✅ | 円単位、見積もり金額でも可 |
| 購入頻度(月次) | プルダウン | ✅ | ✅ | 設定シート参照 |
| 月の発注量見込み | テキスト | ✅ | ✅ | 「不明」ボタンあり |

### 評価

| 項目 | 種類 | 必須 | 自動読込 | 備考 |
|------|------|------|---------|------|
| パートナーシップ度 | プルダウン(1-5) | ✅ | - | 基準に基づき評価 |
| 商談の手応え | プルダウン(1-5) | ✅ | - | |

### パートナーシップ度の基準

| 度 | 基準 |
|----|------|
| 5 | 信頼重視×大口 OR 信頼重視×小口で成長見込みあり |
| 4 | 価格重視×大口 OR 信頼重視×小口 |
| 3 | 個人購入だがまとめ購入（仲間内で共同購入） |
| 2 | 価格重視×小口 |
| 1 | 個人購入×少量（B2C対象外） |

### 「成長見込みあり」の条件

- スタートアップ（新規開業、EC立ち上げ、実店舗オープン予定）
- テスト購入明言（「まずは試しに少量で、良ければ継続」）
- 事業計画あり（「半年後に店舗拡大予定」）
- 資金力の示唆（「予算は確保している」）
- 販路あり（「既に販売先が決まっている」）

### 振り返り

| 項目 | 種類 | 必須 | 条件 |
|------|------|------|------|
| 進捗メモ | 表示のみ | - | 商談管理シートから参照 |
| 商談の手応え | プルダウン(1-5) | ✅ | - |
| 今回の商談で良かった点 | テキスト | ✅ | - |
| こうしたらもっと良かった点 | テキスト | ✅ | 失注時のみ |
| 成約したポイント | テキスト | ✅ | 成約時のみ |
| 次の商談に向けたアクションプラン | テキスト | ✅ | 対象外以外 |
| 次回アクション日 | 日付 | ✅ | 追客時のみ |
| アクション内容 | テキスト | ✅ | 追客時のみ |
| 再アプローチ日 | 日付 | ✅ | 見送り時のみ |
| 見送り理由 | テキスト | ✅ | 見送り時のみ |
| 対象外理由 | テキスト | ✅ | 対象外時のみ |

### 商談ログ

| 項目 | 種類 | 必須 | 備考 |
|------|------|------|------|
| 商談ログ | テキスト | ✅ | 会話ログをコピペ、会話ログシートに格納 |

## 2.3 空白禁止ルール

- 全ての必須項目は空白禁止
- 「不明」と入力する場合は「不明」ボタンを押す
- 空白のまま保存しようとするとエラー表示

## 2.4 保存後の処理

1. 商談レポートシートに保存
2. 商談ログを会話ログシートに保存
3. Gemini APIでログ解析
4. Buddyからフィードバック表示
5. 壁打ち提案

## 2.5 Buddyフィードバック

### 解析内容

| 項目 | 内容 |
|------|------|
| 顧客の反応 | ポジティブ/ネガティブ |
| キーワード抽出 | 重要な発言 |
| 良かった点 | 効果的だったアプローチ |
| 改善点 | 次回に活かせるポイント |

### フィードバック例

```
🧔‍♂️ 商談ログを確認したよ。

📊 【分析結果】
・顧客の反応: ポジティブ（「品質が良さそう」の発言あり）
・キーポイント: 価格ではなく品質を重視している

💡 【良かった点】
・「他社との違いは品質管理です」という説明が刺さったみたい

📝 【次回に活かせるポイント】
・発注量について「まだ決めてない」と言ってたから、
　次回は具体的な提案をしてみるといいかも

💬 よかったら、今回の商談について僕と壁打ちする？

[ 壁打ちする ]  [ 今回はスキップ ]
```

# 5. データ活用

## 5.1 担当者カルテへの反映

- 最新の振り返りを表示
- 振り返り提出率をスコアに反映
- 振り返りの質をGemini APIで分析してスコア化

## 5.2 1on1ミーティングの資料

- 週次/月次レポートをオーナーが参照
- 課題や目標の推移を確認
- 具体的なアドバイスの材料に
